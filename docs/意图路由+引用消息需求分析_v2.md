# 意图路由 + 引用消息处理 — 需求分析 v2

## 核心约束

> **路由只分 4 类**：识图 / 存图 / 发图 / 正常对话
> 
> **路由后的数据流**：`[INTENT:xxx]` + 原始消息文本 → 传给 OpenClaw
> 
> **引用内容先缓存，再路由**：引用消息中的图片、合并转发内容，**在路由之前全部缓存到图床和本地**
> 
> **提示词拼接仍保留**，但只拼群 prompt + 上下文，不再拼大段技能指令

---

## 1. 意图路由 — 简洁版

### SF 系统提示词

```
你是一个QQ群聊消息的意图分类器。
分析用户消息，只输出一个词：vision(识图) save_img(存图) send_img(发图) chat(正常对话)

规则：
- 用户想看/识别图片 → vision
- 用户想保存图片到本地 → save_img
- 用户想让你发一张图片 → send_img
- 其他 → chat

消息属性：用户消息含图片={has_image}，引用了图片={has_image_ref}，引用了合并转发={has_forward_ref}
```

## 2. 引用消息 — 桥接层预处理（路由前）

**不要把引用逻辑塞进路由层**，桥接在消息解析阶段就处理好引用信息，传给后续流程用。

### 先缓存，再路由

**缓存规则**：只缓存到**图床 tc/1/**（临时目录），`\转发图片\` 是永久收藏，只有 save_img 才写进去。

**用完清理**：非 save_img 的引用缓存，handler 处理完后删除。

```
NapCat 消息
  ↓
消息解析
  ├─ text = 用户回复文本
  ├─ img_urls = 用户本条带的图片
  └─ 检测引用消息（if rid）
       ├─ get_msg(message_id) → 获取引用消息内容
       │    └─ 含图片 → 下载到图床 tc/1/（临时缓存）
       ├─ 是合并转发 → 展开转发内容
       │    ├─ 嵌套转发 → 标记 [嵌套转发(无法展开)]
       │    └─ 提取所有图片URL → 下载到图床 tc/1/（临时缓存）
       └─ 组装 ref_context
            ├─ type: "image" | "forward" | "text" | null
            ├─ cached_count: 本次缓存的图片数量
            ├─ forward_img_count: 转发中图片总数
            └─ cached_paths: [图床缓存文件路径]

  ↓
意图路由（SF 分类 4 桶）
  ↓
按意图分流 → handler
                 │
                 ├─ save_img → 从 tc/1/ 复制到 \转发图片\（永久）→ 删 tc/1/ 缓存
                 ├─ vision   → 从 tc/1/ 取图 → SF vision → 删 tc/1/ 缓存
                 ├─ send_img → 从 tc/1/ 取图 → 发图 → 删 tc/1/ 缓存
                 └─ chat     → 删 tc/1/ 缓存（没用上）
```

### RefContext 不参与路由判断

```
路由层看到的只是：
  text = "[引用消息] 存这张"    (引用图片)
  has_image = false
  has_forward_ref = false

或：
  text = "[引用合并转发] 帮我看看"  (引用转发)
  has_image = false
  has_forward_ref = true

路由只认意图 → 输出 intent + 原文本 → 传给 OC
```

### 引用数据传给 OC 的方式

```
传给 OC 的消息格式：

[INTENT:save_img] [引用图片] [QQ:123456] 存这张

OC 从 [引用图片] 标记就知道用户引用了一张图，但无需知道具体路径。
桥接收到 OC 回复后，根据 INTENT + 引用信息执行具体操作。
```

## 3. 各意图的分流后操作

路由映射到 4 个 handler，每个 handler 桥接层自己决定操作 + 传 OC 生成回复语。

### 3.1 vision（识图）

**桥接操作**：
- 图片来源：缓存中找（引用图片 / 用户发的图 / 转发缓存），**都已在本地图床**
- 选图 → SF vision 描述

**传给 OC**：
```
[INTENT:vision] [QQ:uid] 用户消息文本
[SF描述: 图片中有...]
```

**OC 的职责**：润色描述 + 自然回复

**桥接收回复后**：原文本发回群聊

### 3.2 save_img（存图）

**桥接操作**：
- 引用图片已缓存到图床 **tc/1/**（临时）
- 从 tc/1/ 复制到桌面 `\转发图片\`（或子文件夹）—— 只有 save_img 才写永久目录
- 删 tc/1/ 缓存

**传给 OC**：
```
[INTENT:save_img] [QQ:uid] 用户消息文本
```

**OC 的职责**：回复"存好啦" / 问存到哪个文件夹

### 3.3 send_img（发图）

**桥接操作**：
- 引用转发缓存已在 tc/1/ → 直接 send_image.py 发图 → 删缓存
- 普通发图 → 从转发图片目录选图（pick_fwd_image）→ 复制到 tc/1/ → 发图

**传给 OC**：
```
[INTENT:send_img] [QQ:uid] 用户消息文本
```

**OC 的职责**：回复配图语，或纯发图不说话

### 3.4 chat（正常对话）

**桥接操作**：删 tc/1/ 缓存（引用内容没用上）

**传给 OC**：
```
[INTENT:chat] [QQ:uid] 用户消息文本
```

**OC 的职责**：正常聊天

async def _handle_vision(self, gid, uid, text, img_urls, ref_context, at_bot):
    # 图片来源优先级：引用图片 > 用户发的图 > 转发缓存
    target_images = []
    if ref_context and ref_context.images:
        target_images = ref_context.images  # 引用图片
    elif img_urls:
        target_images = img_urls  # 用户发的图
    elif ref_context and ref_context.forward_images:
        # 引用转发 → 取第N张 或 全部
        target_images = ref_context.forward_images
    
    # SF vision 描述
    sf_desc = await self._vision_raw(target_images[0])
    
    # 传给 OC
    oc_input = f"[INTENT:vision] [QQ:{uid}] {text}\n[SF描述: {sf_desc}]"
    reply = await self._call_oc(..., oc_input)
    await self._send_reply(gid, reply)
