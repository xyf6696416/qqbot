## 1. SF 意图分类系统提示词

### 路由层的输入

```
{
  "text": "用户消息文本（含 [引用消息] 等桥接标记）",
  "has_image": true/false,          // 用户本条消息是否带了图片附件
  "has_ref": true/false,            // 是否引用/回复了某条消息
  "ref_type": null/"text"/"image"/"forward",  // 引用的消息类型
  "ref_text": "引用的消息内容（若 ref_type=text/forward）",
  "ref_img_count": 3                // 引用中的图片数量
}
```

### 候选系统提示词 v1（关键词辅助 + chat 兜底）

```
你是一个QQ群聊机器人桥接层的意图分类器。
你的任务是分析用户消息的意图，输出JSON格式结果，不要输出其他内容。

## 意图定义

| 意图 | 场景 | 示例 |
|------|------|------|
| chat | 普通聊天、问问题、闲聊、对话题评论 | "今天好热" "晚上吃啥" "你刚才说的不对" |
| send_img | 用户想让机器人发一张/几张本地图片 | "发张图" "来点萝莉" "发两张脚的" |
| vision | 用户想看/识别一张图片（含引用图） | "这是啥" "图里有什么" "帮我看看" |
| save_img | 用户想保存图片到本地文件夹 | "存起来" "保存这张" "存到萝莉" |

## 规则（按优先级）

1. 如果消息含图片附件（has_image=true），优先判断 vision 或 save_img
2. 如果消息是引用（has_ref=true），检查被引用的内容类型（ref_type）：
   - ref_type=image + 含"存"/"保存" → save_img
   - ref_type=image + 含"发"/"来"/"类似" → send_img
   - ref_type=image + 识图相关 → vision
   - ref_type=forward + 含"存"/"保存" → save_img
   - ref_type=forward + 容"发"/"看看" → send_img 或 vision
3. 文本中含"发图"/"来张"/"来点"/"色图" → send_img（提取 count 和 src）
4. 文本中含"存图"/"保存"/"存起来" → save_img（提取子文件夹名）
5. 以上都不命中 → chat

## 提取参数

- count: 发图数量（从"两张"/"三个"/"N张"中提取，默认1）
- src: 发图子文件夹，从消息中匹配关键词如 萝莉/脚/大雷/同人/色彩图/轮奸
- save_to: 存图的目标子文件夹名，从"存到XXX"中提取
- ref_action: 对引用的操作(null/identify/save/resend)

## JSON 输出格式

{"intent":"chat|send_img|vision|save_img","count":1,"src":null,"save_to":null,"ref_action":null,"reason":"简短原因"}

## 示例

用户消息: "发张萝莉图看看"
{"intent":"send_img","count":1,"src":"萝莉","save_to":null,"ref_action":null,"reason":"明确发图请求"}

用户消息: "[引用消息] 保存这张"
(假设 ref_type=image)
{"intent":"save_img","count":1,"src":null,"save_to":null,"ref_action":"save","reason":"引用图片要求保存"}

用户消息: "[引用合并转发] 里面第三张帮我看看"
(假设 ref_type=forward, ref_img_count=5)
{"intent":"vision","count":0,"src":null,"save_to":null,"ref_action":"identify","reason":"引用转发中的图片要求识别"}
```

### 优化建议：这份 prompt 太长

考虑到 SF 调用有延迟，prompt 越长 → 推理越慢 → tokens 越多。

**实测**: 上述完整 prompt 约 900 tokens input, 每次输出约 50 tokens

**改进**: 用更短的 prompt + 硬编码规则在 bridge 层处理粗分类，SF 只做模糊判断

### 精简版 prompt

```
你是QQ群聊消息的意图路由器。分析用户消息，输出JSON。

消息属性: has_image={has_image}, has_ref={has_ref}, ref_type={ref_type}, ref_img_count={ref_img_count}

意图: chat(聊天) | send_img(发图) | vision(识图/看图片) | save_img(存图)

规则:
- has_image=true + 识图词 → vision
- has_image=true + "存"/"保存" → save_img  
- ref_type=forward + "存"/"保存" → save_img
- ref_type=forward + "发"/"看看" → send_img
- 消息含"发图"/"来张"/"色图"/"发.{0,5}图" → send_img
- 消息含"存图"/"保存" → save_img
- 其他 → chat

从消息中提取: count(数量,如"两张"=2,默认1), src(子文件夹,如"萝莉"/"脚"/"大雷"/null), save_to(存图目标文件夹/null)

只输出JSON: {"intent":"","count":1,"src":null,"save_to":null,"ref_action":null}
```

## 2. 引用消息的完整场景分析

### 2.1 当前引用处理（bridge.py 现有逻辑）

```python
if rid:  # 用户回复/引用了某条消息
    q_msg = get_msg(message_id)
    q_imgs = extract_imgs(q_msg)  # 提取图片URL
    img_urls = q_imgs + img_urls  # 合并引用图 + 用户自己发的图
    
    # 检测引用的是不是合并转发
    if quoted_forward:
        expand forward → text + img cache
        text = "[引用合并转发]\n{forward_text}\n[用户回复] {text}"
    else:
        text = "[引用消息] " + text
```

**当前问题**：
1. 引用图片和回复图片混在一起，无法区分
2. 引用合并转发时，所有图片全部缓存到图床，但 agent 不知道用户到底想干嘛
3. 没有区分引用不同类型消息的意图差异

### 2.2 完整引用场景矩阵

| # | 引用类型 | 用户说的话 | 真实意图 | 路由后操作 | 
|---|---------|-----------|---------|-----------|
| 1 | 图片 | "这是啥" / "是什么" | **vision** | SF vision 描述图片 → OC 润色 |
| 2 | 图片 | "保存" / "存起来" | **save_img** | 复制引用图片到桌面 `\转发图片\` |
| 3 | 图片 | "存到萝莉" | **save_img** | 复制到 `\转发图片\萝莉\` |
| 4 | 图片 | "好看" / "不错" | **chat** | 直接 OC 聊天 |
| 5 | 图片 | "发几张类似的" | **send_img** | 从同类型文件夹选类似图发 |
| 6 | 文本 | "你说的不对" | **chat** | 直接 OC 聊天 |
| 7 | 文本 | "什么意思" | **chat** | OC 结合上下文解释 |
| 8 | 合并转发 | "存图" | **save_img** | 将转发中所有图片存到桌面 |
| 9 | 合并转发 | "存到萝莉" | **save_img** | 存到子文件夹 |
| 10 | 合并转发 | "发出来看看" | **send_img** | 从缓存图床重新发到群里 |
| 11 | 合并转发 | "第三张帮我看看" | **vision** | 定位转发中第3张图，SF vision |
| 12 | 合并转发 | "这里面都是啥" | **chat** | 描述转发内容（OC 可做） |
| 13 | 无引用 | "发张萝莉" | **send_img** | 选图 → 发图 |
| 14 | 无引用 | "今天天气" | **chat** | OC 聊天 |

### 2.3 引用处理中的技术难点

**难点 A：引用合并转发中的单张图**（场景 #11）

用户说"第三张帮我看看"，桥接需要：
1. 知道转发中有哪些图片（已缓存到图床，知道路径）
2. 定位第3张图
3. 只对该图做 vision
4. 其他图不管

**当前能做到的**：`_cache_fwd_images` 已经把转发中的图片下载到图床，按时间顺序命名 `fwd_{ts}_{idx}.jpg`
**缺少的**：序号 ↔ 图片的映射关系（idx 是全局自增不是转发内的排序）

**难点 B：引用转发中的批量存图**（场景 #8）

用户说"存图"，桥接需要：
1. 知道引用的是哪个转发消息（已知 `rid`）
2. 转发中包含的图片都在图床缓存（已做到）
3. 复制到桌面 `\转发图片\`
4. 可以按子文件夹分类

**难点 C：引用图片发类似图**（场景 #5）

"发张类似的"需要：
1. 知道引用图片的内容（SF vision 描述）
2. 根据描述关键词从本地库找图
3. 选图发送

**这个实现成本较高**，可以先标记为"未来支持"

### 2.4 引用信息的数据结构

桥接解析阶段需要提取的引用信息：

```python
class RefInfo:
    type: str              # "text" | "image" | "forward" | "mixed"
    images: list[str]      # 引用消息中的图片 URL
    text: str              # 引用消息中的纯文本
    is_forward: bool       # 是否是合并转发
    forward_nodes: list    # 展开后的转发节点列表
    forward_images: list   # 转发中所有图片的缓存路径（已被 _cache_fwd_images 下载）
    forward_image_count: int  # 转发中的图片数量
    cached_paths: list[str]   # 图床缓存路径（tc/1/ 下的文件）
```

## 3. 意图路由 + 引用的结合方案

### 3.1 数据流

```
NapCat 消息
  ↓
消息解析层（现有）
  ├─ 提取文本 text
  ├─ 提取图片 img_urls
  ├─ 提取引用（rid → get_msg）
  │    ├─ 识别引用类型（text/image/forward）
  │    ├─ 展开合并转发 → forward_nodes
  │    ├─ 缓存转发图片 → cached_paths
  │    └─ 组装 ref_info
  └─ 组装 router_input
  ↓
意图路由层（新增）
  ├─ [关键词匹配] 快速判断
  │    ├─ 命中 → 直接分流
  │    ├─ 模糊 → SF 分类（精简版 prompt）
  │    └─ 未命中 → chat
  └─ 返回 IntentResult
  ↓
按意图分流处理
  ├─ chat → 精简 prompt + OC API
  ├─ send_img → pick_fwd → copy → send_image
  │               └─ 可选: 引用影响（ref_type=forward→从缓存发）
  ├─ vision → SF vision → OC 润色
  │               └─ 可选: 引用影响（ref_type=forward→定位特定图）
  └─ save_img → copy to desktop
                  └─ 引用影响（ref_type=forward→批量存）
```

### 3.2 引用路由决策树

```
用户引用了消息？
  ↓
是 → 判断引用类型
      │
      ├─ image（引用图片）
      │   ├─ 用户说"存"/"保存" → save_img（引用图片存桌面）
      │   ├─ 用户说"这是啥"/"看看" → vision（识图）
      │   ├─ 用户说"类似的" → send_img（基于图片关键词推荐）
      │   └─ 其他 → chat
      │
      ├─ text（引用文本）
      │   └─ 全部 → chat（带上引用上下文）
      │
      ├─ forward（引用合并转发）
      │   ├─ 用户说"存" / "保存" → save_img（批量存转发中所有图）
      │   │    参数: save_to = 从"存到XXX"提取子文件夹名
      │   ├─ 用户说"第N张看看" → vision（定位第N张识别）
      │   │    参数: target_index = N
      │   ├─ 用户说"发出来" / "看看" → send_img（从缓存重发图片）
      │   │    参数: count = 转发中图片数
      │   ├─ 用户说"第N张存" → save_img（单张存）
      │   │    参数: target_index = N, save_to=...
      │   └─ 其他 → chat
      │
      └─ mixed（引用包含图文）
          └─ 同 image（图片优先）
  ↓
否 → 正常意图路由
      ├─ "发图"/"来张" → send_img
      ├─ "存图"/"保存" → save_img
      ├─ 含图片附件 → vision
      └─ 其他 → chat
```

### 3.3 优化建议：路由规则硬编码 + SF 只做模糊处理

基于上述决策树，绝大多数引用场景可以**硬编码**：

```python
def route_with_ref(text, has_image, ref_info):
    """硬编码引用路由规则"""
    # ===== 无引用 =====
    if not ref_info:
        # 简单关键词匹配
        if any(kw in text for kw in ["发图","来张","来点","发色","色图"]):
            return Intent("send_img", extract_count_src(text))
        if any(kw in text for kw in ["存图","保存","存起来","存到"]):
            return Intent("save_img", extract_save_to(text))
        if has_image:
            return Intent("vision")
        return Intent("chat")
    
    # ===== 引用图片 =====
    if ref_info.type == "image":
        if any(kw in text for kw in ["存","保存","存到"]):
            return Intent("save_img", save_to=extract_save_to(text), 
                         ref_action="save", images=ref_info.images)
        if any(kw in text for kw in ["是啥","什么","看看","这","图里","识别"]):
            return Intent("vision", ref_action="identify", images=ref_info.images)
        return Intent("chat")
    
    # ===== 引用合并转发 =====
    if ref_info.type == "forward":
        # "第N张" 检测
        nth_match = re.search(r'第(\d+)张', text)
        if nth_match:
            idx = int(nth_match.group(1))
            if idx <= len(ref_info.forward_images):
                target_img = ref_info.forward_images[idx - 1]
                if any(kw in text for kw in ["存","保存"]):
                    return Intent("save_img", save_to=extract_save_to(text),
                                 target_index=idx, images=[target_img])
                else:
                    return Intent("vision", target_index=idx, images=[target_img])
        # "存图" / "保存" → 批量存
        if any(kw in text for kw in ["存","保存","存起来"]):
            return Intent("save_img", images=ref_info.forward_images,
                         save_to=extract_save_to(text))
        # "发出来" / "看看" → 重发
        if any(kw in text for kw in ["发","看看","出来"]):
            return Intent("send_img", count=len(ref_info.forward_images),
                         images=ref_info.forward_images)
        return Intent("chat")
    
    return Intent("chat")
```

**SF 只负责**：关键词无法明确判断的模糊 case
- "来点好康的" → 关键词不匹配，走 SF
- "你看这个" → 关键词不匹配，走 SF
- "整一个" → 关键词不匹配，走 SF

### 4.3 改后 _process_queue_item 结构

```python
async def _process_queue_item(self, item):
    gid, uid, text, at_bot = item["gid"], item["uid"], item["text"], item["at_bot"]
    img_urls = item.get("img_urls", [])
    ref_info = item.get("ref_info")  # 新增：引用信息
    
    # ===== Step 1: 意图路由 =====
    intent = self._route_intent(text, has_image=bool(img_urls), ref_info=ref_info)
    log.info("INTENT: %s (src=%s, count=%d, ref_action=%s, source=%s)",
             intent.intent, intent.src, intent.count, intent.ref_action, intent.source)
    
    # ===== Step 2: 按意图分流 =====
    if intent.intent == "send_img":
        return await self._handle_send_img(gid, uid, text, intent, at_bot)
    elif intent.intent == "vision":
        # vision 的图片来源：优先引用图，次选用户发的图
        vision_images = intent.ref_images or img_urls
        return await self._handle_vision(gid, uid, text, vision_images, at_bot)
    elif intent.intent == "save_img":
        return await self._handle_save_img(gid, uid, text, intent, at_bot)
    elif intent.intent == "chat":
        return await self._handle_chat(gid, uid, text, intent, at_bot)

def _route_intent(self, text, has_image, ref_info) -> IntentResult:
    """意图路由：关键词匹配"""
    # Step A: 硬编码关键词规则
    result = self._keyword_route(text, has_image, ref_info)
    if result.confidence == "high":
        return result
    
    # Step B: 模糊 case 走 chat
    if result.confidence in ("medium", "low"):
        return IntentResult("chat", "fallback", {})
        # SF 失败时保留关键词的结果
        return result
    
    return result
```

## 5. 总结

1. **引用消息需要桥接层解析并结构化**：提取 ref_type / images / text / forward_nodes 等字段
2. **意图路由决策树**：引用图片/文本/合并转发各有不同的处理路径
3. **86% 的引用 case 可以用硬编码覆盖**，SF 只做模糊兜底
4. **发图路由**：无引用走 `pick_fwd_image`，引用转发走缓存重发
5. **存图路由**：非引用暂不处理（等待用户主动引），引用直接复制
6. **识图路由**：用户图片 + 引用图片都支持，引用转发支持"第N张"

需要评审的部分：
- **SF prompt 精简版**（第1节）
- **引用路由决策树**（第3.2节）
- **硬编码路由规则**（第3.3节）
